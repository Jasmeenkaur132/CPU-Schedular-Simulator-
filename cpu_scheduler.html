01

Algorithm theory

FCFSNon-preemptive

Processes run strictly in the order they arrive.

The simplest algorithm. The CPU is assigned to processes in arrival order. Once started, a process runs to completion — no interruptions allowed.

Pros
  • Simple to implement
  • Fair by arrival order
  • No starvation
Cons
  • Convoy effect
  • High avg wait time
  • Not optimal
SJFNon-preemptive

Picks the arrived process with the shortest burst time next.

Among all arrived processes, the shortest burst job runs next. Optimal average waiting time among non-preemptive algorithms.

Pros
  • Minimizes avg wait
  • Optimal (non-preempt)
  • Good throughput
Cons
  • Starvation possible
  • Burst time unknown
  • Not fair to long jobs
SRTFPreemptive

Preemptive SJF — always runs the shortest remaining job.

At every tick, the process with the shortest remaining time runs. A new arrival can preempt the current process if it has less remaining time.

Pros
  • Optimal avg wait
  • Fast short processes
  • Very responsive
Cons
  • Heavy context switch
  • Long job starvation
  • Complex to implement
Round RobinPreemptive

Each process gets a fixed time quantum in circular rotation.

Processes share the CPU in turns. Each gets a time quantum; if unfinished, it rejoins the back of the queue. Designed for time-sharing fairness.

Pros
  • Fair to all processes
  • No starvation
  • Good for time-sharing
Cons
  • High context switches
  • Quantum choice matters
  • High avg turnaround
